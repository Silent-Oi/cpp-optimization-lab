# 项目路线图

本路线图说明各子项目为什么按当前顺序出现，以及每个项目需要为后续阶段留下什么能力。路线允许根据实际学习情况调整；只有当前项目和紧邻的下一项目需要详细规划。

## 能力递进总览

| 阶段 | 计算模式 | 主要新增能力 | 为后续提供的基础 |
|---|---|---|---|
| Project00 — Common | 公共基础设施 | CMake 组织、共享接口、实验工具复用边界 | 所有项目的构建与测量基础 |
| Project01 — Batch Oscillator | 大量相互独立的状态更新 | 数值基线、连续内存、AoS/SoA、基础 benchmark、自动向量化入门 | 批量计算、正确性与性能并重的实验习惯 |
| Project02 — 2D Heat Equation | 规则网格与 stencil | 边界条件、稳定性、收敛、空间局部性、循环顺序 | PDE 离散和网格性能模型 |
| Project03 — Monte Carlo Transport | 随机独立任务 | RNG 状态、统计误差、可复现随机性、分支和并行负载 | 随机算法与多线程实验方法 |
| Project04 — Sparse Poisson Solver | 稀疏线性代数 | CSR、SpMV、CG、简单预条件、matrix-free | 中期目标；科学计算求解器基础 |
| Project05 — Basic CPU Ray Tracer | 几何求交与高分支负载 | 向量几何、相机、求交、材质、图像正确性 | 渲染管线和空间加速结构的输入问题 |
| Project06 — N-body / Barnes–Hut | 空间层次结构 | 树构建、近似误差、复杂度、空间数据布局 | BVH 构建与遍历所需的树结构经验 |
| Project07 — Multithreaded CPU Path Tracer | 综合不规则负载 | 三角形网格、BVH、随机采样、任务调度、综合性能分析 | 最终综合目标 |

## Project00 — Common

定位：持续演进的公共基础层，不是独立物理项目。

只在以下情况加入组件：

- 至少两个项目已经需要同一能力；或
- 明确属于通用实验基础设施，例如计时、环境信息和浮点比较。

不提前建设通用数学库、线程池或复杂模板框架。

## Project01 — Batch Oscillator

定位：第一个完整的“数学—实现—验证—测量—优化”项目。

本项目分为两层：

- 无阻尼简谐振子用于学习解析解、时间积分、相位误差和能量误差。
- 批量线性阻尼振子用于连续内存、数据布局和 CPU 性能实验。

核心学习结果：

- 建立标量正确性参考和批量 baseline。
- 形成最小但可信的 benchmark 流程。
- 使用瓶颈证据决定是否进行数据布局、向量化或多线程实验。
- 区分物理耗散和数值误差。

RK4、完整积分器谱系、手动 SIMD 和复杂线程基础设施不是核心完成条件；只有满足各自进入条件时才作为可选实验。

## Project02 — 2D Heat Equation

定位：从独立元素更新过渡到具有邻域依赖的规则网格计算。

重点新增：

- 空间离散、边界条件和显式时间推进稳定性。
- 二维数组的线性存储、按行访问和 stencil 缓存局部性。
- 正确性、收敛性和性能之间的联合验证。

进入条件：Project01 已建立可复用的正确性检查和 benchmark 习惯。

## Project03 — Monte Carlo Transport

定位：学习随机算法以及工作量不完全均匀的并行任务。

重点新增：

- 随机数状态与可复现性。
- 统计误差、样本规模和置信度。
- 分支行为、线程本地状态和负载均衡。

不在此阶段提前建设通用任务系统；先使用最简单的可测并行方案。

## Project04 — Sparse Matrix and Poisson Solver

定位：中期目标，把网格离散、线性代数、内存访问和迭代算法连接起来。

核心范围：

- 二维 Poisson 方程离散。
- 稠密参考、CSR 稀疏存储和 matrix-free 三种表达。
- SpMV、基础向量运算、共轭梯度法和简单预条件。
- 残差、收敛历史、内存占用和性能对比。

完成后应能解释算法复杂度、存储格式、缓存行为和收敛速度之间的关系。

## Project05 — Basic CPU Ray Tracer

定位：建立正确、可验证的 CPU 渲染基线。

重点新增：

- 向量几何、光线、相机和几何求交。
- 简单材质与图像输出。
- 高分支程序的正确性和性能分析。

第一版不追求复杂材质、网格优化或完整路径追踪。

## Project06 — N-body and Barnes–Hut

定位：在进入 BVH 前独立学习空间层次结构。

重点新增：

- 从 \(O(N^2)\) baseline 到层次近似算法。
- 树构建、遍历、近似误差和数据布局。
- 并行构建或遍历中的负载不均衡。

## Project07 — Multithreaded CPU Path Tracer

定位：最终综合项目。

核心范围：

- 三角形网格、包围盒、BVH 构建与遍历。
- 随机采样、光线多次反弹和结果收敛。
- 图像分块、线程调度与负载均衡。
- 几何布局、树节点布局、随机数状态、分支和内存带宽分析。

项目完成标准不仅包括正确图像和更高性能，还包括可以复现的实验记录，以及对主要瓶颈和工程权衡的解释。
