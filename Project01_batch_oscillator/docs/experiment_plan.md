# Project01 实验计划

本文只描述 Project01 特有的实验顺序和假设。通用测量字段、原始数据表和结论格式使用根目录 [`docs/benchmark_record_template.md`](../../docs/benchmark_record_template.md)。

## 实验进入顺序

```text
正确性门槛
    ↓
AoS baseline 与规模实验
    ↓
初步瓶颈判断
    ↓
AoS / SoA 与自动向量化实验
    ↓
按证据选择预计算对比或简单多线程
```

不得为了执行后面的实验而跳过前面的正确性或测量门槛。

## 共同正确性门槛

每个性能版本计时前都必须满足：

- 使用相同物理模型、输入数据、时间步长和时间步数。
- 与标量参考结果在约定容差内一致。
- 保留相同含义的校验值。
- 初始化和验证不进入核心循环计时。
- 若结果含义或算法发生变化，不能把差异全部归因于性能优化。

## E0 — 无阻尼长期行为记录

性质：数值学习实验，不是正式性能 benchmark。

目的：用最小实验收尾已有积分器学习。

固定：

- 相同 `omega`、初始状态、`dt` 和总时间。

比较：

- 显式 Euler。
- 辛 Euler。
- 一种 Verlet。

至少记录：

- 位置或相空间状态。
- 能量随时间的代表性变化。
- 相位误差与能量误差不能互相替代的观察。

完成后不要求继续实现 RK4 或完整积分器收敛阶实验。

## E1 — AoS baseline 规模实验

进入条件：M2 批量结果已通过标量回归。

第一条假设：

> 当 batch 超过缓存容量后，每秒更新数会出现不同于小规模数据的变化，因为核心循环的数据需要从更低层缓存或主内存读取。

建议规模从以下数量级选择并根据实际运行时间调整：

```text
1,000
10,000
100,000
1,000,000
```

固定：

- AoS 布局。
- 预计算精确一步更新。
- 相同随机种子、`dt` 和时间步数。
- Release 构建和单线程。

输出：

- 总时间。
- 每秒振子更新数。
- 每振子每步耗时。
- 校验值。
- 原始重复测量和波动。

本实验结束后估计每次更新的读取/写入字节数和主要浮点运算，形成初步瓶颈判断。

## E2 — AoS 与 SoA

进入条件：E1 测量稳定，且两个布局已通过一致性验证。

假设：

> SoA 是否更快取决于核心循环访问的字段、数据规模和编译器向量化结果，而不是布局名称本身。

唯一主要改变：数据组织方式。

必须固定：

- 输入数据。
- 更新方程和预计算系数含义。
- `dt`、时间步数和规模。
- 编译选项和线程数。
- 校验方式。

至少比较一组缓存内规模和一组超过缓存的规模，并结合自动向量化报告解释结果。

## E3 — 自动向量化

进入条件：已经得到 AoS/SoA 稳定结果。

问题：

1. 编译器是否向量化核心循环？
2. 如果没有，阻碍因素是别名、控制流、布局还是其他依赖？
3. 最小结构调整是否改变生成代码和实际性能？

证据优先级：

1. 编译器向量化报告。
2. 受控性能变化。
3. 必要时查看核心循环的最小汇编范围。

不因为看到 SIMD 指令就自动认定程序更快，也不在此阶段默认进入手动 SIMD。

## E4 — 实时计算与预计算系数（条件实验）

进入条件：E1/E2 表明计算与参数读取之间的权衡值得单独研究。

比较：

- 热循环实时根据原始参数计算更新所需系数。
- 初始化阶段预计算并在热循环读取系数。

该实验可能同时改变计算量、参数读取量和初始化成本。报告必须分别说明：

- 核心循环性能。
- 初始化成本是否计入目标使用场景。
- 两种方式的结果误差是否一致。
- 结论是否随时间步数和 batch 规模变化。

## E5 — 简单多线程分块（条件实验）

进入条件：

- 单线程 benchmark 稳定。
- 目标规模足够大。
- 各振子完全独立。
- 已明确为什么并行实验具有学习价值。

第一版只做连续索引静态分块，不建设线程池。

至少测试：

- 1 线程。
- 2 线程。
- 4 线程。
- 物理核心数附近。

同时记录正确性、加速比、并行效率和大规模下的内存带宽迹象。若不满足进入条件，记录原因即可，不影响 Project01 核心完成。

## 可选但不预排期

- Debug 与 Release 的一次教学性观察。
- RK4 或完整收敛阶实验。
- 手动 SIMD。
- 临界阻尼和过阻尼。

这些项目只有在新的学习问题出现时才建立正式实验记录。

## 记录位置

- 原始数据和临时输出：`results/`，默认不提交 Git。
- 可复现实验报告：`docs/experiments/`，提交 Git。
- 每份正式报告只回答一个主要假设。
