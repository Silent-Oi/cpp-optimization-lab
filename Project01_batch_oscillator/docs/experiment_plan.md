# Project01 性能实验计划

## 实验目标

通过批量线性阻尼振子模拟，理解数据布局、循环组织和编译优化对 CPU 性能的影响。
无阻尼简谐振子保留为数值方法学习和回归检查的基线。

## 固定条件

每轮正式对比前记录：

- 编译器：
- 编译器版本：
- 构建类型：
- CPU：
- 操作系统：
- 物理模型：
- 振子数量：
- 固有角频率范围：
- 阻尼参数表达与范围：
- 时间步数：
- 时间步长：
- 数值方法：
- 随机种子或初始条件：
- 预热次数：
- 测量次数：

## 通用测量流程

1. 先用独立的小规模输入验证结果正确性。
2. 使用 Release 构建，并记录实际执行的构建命令和运行命令。
3. 正式计时前执行预热，避免首次运行成本混入结果。
4. 每个版本交替或随机顺序运行多次，记录每次原始结果，不只记录最快值。
5. 报告计时单位和中位数；必要时同时记录最小值、最大值或离散程度。
6. 每次计时都保留同一含义的校验值，防止计算被编译器消除或结果已经发生变化。

## 实验 A：Debug 与 Release

目的：观察编译优化对数值循环的影响。

保持不变：

- 相同源码
- 相同输入规模
- 相同输出校验方式

改变：

- 构建类型

## 实验 B：AoS 与 SoA

目的：观察不同数据布局对连续批量计算的影响。

保持不变：

- 数值方法
- 输入数据
- 时间步数
- 计算结果含义

改变：

- 数据组织方式

## 实验 C：问题规模

建议测试多个数量级，例如：

```text
1,000
10,000
100,000
1,000,000
```

实际规模根据内存和运行时间调整。

## 实验 D：循环内重复计算

检查：

- 是否重复计算不随循环变化的量
- 是否存在不必要的对象创建
- 是否存在可提前计算的系数
- 是否重复计算阻尼项系数
- 是否存在频繁的边界检查或函数调用

## 结论要求

每次对比至少回答：

1. 哪个版本更快？
2. 快多少？
3. 结果是否仍然正确？
4. 差异是否随问题规模变化？
5. 你认为主要原因是什么？
6. 是否存在其他可能解释？
7. 多次测量的波动是否足以影响结论？
