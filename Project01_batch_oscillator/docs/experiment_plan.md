# Project01 性能实验计划

## 实验目标

通过批量简谐振子模拟，理解数据布局、循环组织和编译优化对 CPU 性能的影响。

## 固定条件

每轮正式对比前记录：

- 编译器：
- 编译器版本：
- 构建类型：
- CPU：
- 操作系统：
- 振子数量：
- 时间步数：
- 时间步长：
- 数值方法：
- 测量次数：

## 实验 A：Debug 与 Release

目的：观察编译优化对数值循环的影响。

保持不变：

- 相同源码
- 相同输入规模
- 相同输出校验方式

改变：

- 构建类型

## 实验 B：AoS 与 SoA

目的：观察不同数据布局对连续批量计算的影响。

保持不变：

- 数值方法
- 输入数据
- 时间步数
- 计算结果含义

改变：

- 数据组织方式

## 实验 C：问题规模

建议测试多个数量级，例如：

```text
1,000
10,000
100,000
1,000,000
```

实际规模根据内存和运行时间调整。

## 实验 D：循环内重复计算

检查：

- 是否重复计算不随循环变化的量
- 是否存在不必要的对象创建
- 是否存在可提前计算的系数
- 是否存在频繁的边界检查或函数调用

## 结论要求

每次对比至少回答：

1. 哪个版本更快？
2. 快多少？
3. 结果是否仍然正确？
4. 差异是否随问题规模变化？
5. 你认为主要原因是什么？
6. 是否存在其他可能解释？
