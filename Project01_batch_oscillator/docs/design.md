# Project01 设计记录

## 1. 问题定义

在这里记录项目最终采用的物理模型、变量含义和输入参数。

需要明确：

- 振子数量
- 每个振子的状态
- 每个振子的参数是否相同
- 时间步长
- 总时间步数
- 初始条件
- 数值积分方法

## 2. 正确性标准

说明如何判断实现是正确的。

可考虑：

- 与解析解比较
- 能量误差
- 特定初始条件下的周期性
- 不同实现之间的结果差异
- 浮点误差允许范围

## 3. 数据布局

### 基线方案

记录第一版数据结构。

### AoS

记录每个对象包含哪些成员，以及内存中的大致排列方式。

### SoA

记录每个数组保存什么，以及为什么可能改善连续访问。

## 4. 模块划分

在写代码前先简单列出计划模块，不必过度设计。

示例类别：

- 状态数据
- 时间推进
- 参数配置
- 结果校验
- benchmark 驱动
