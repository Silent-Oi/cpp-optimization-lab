# Project01 — Batch Oscillator

## 项目简介

Project 1 以大量相互独立的阻尼振子为计算对象，
最终目标是实现一个能够高效更新约百万个具有不同位移、速度和物理参数的振子状态的 C++ 程序，
并在保证计算结果正确的前提下，通过连续内存、AoS/SoA 数据布局、减少复制与内存分配、编译器向量化和基础多线程等优化，
研究数据组织与 CPU 执行效率之间的关系，建立后续高性能数值计算项目所需的性能测量和底层理解。

## 物理模型

Project01 的最终模型是带线性黏性阻尼的一维振子：

```text
m x'' + c x' + k x = 0
```

使用固有角频率 \(\omega=\sqrt{k/m}\) 和阻尼比
\(\zeta=c/(2\sqrt{km})\) 后，可写成一阶系统：

```text
dx/dt = v
dv/dt = -2*zeta*omega*v - omega^2*x
```

第一版以欠阻尼情形 `0 < zeta < 1` 为主；`zeta = 0` 退化为已经学习和实现的简谐振子，
用于回归检查。临界阻尼和过阻尼可作为后续扩展，不进入第一版核心范围。

前期的简谐振子解析解、显式 Euler、辛 Euler 和 Verlet 实验继续保留，作为学习数值积分、
相位误差和数值能量误差的基础。进入阻尼模型后，加速度同时依赖位置和速度，需重新确认
积分方法是否适用，并区分真实的物理耗散与数值方法引入的额外耗散或增能。

- 阶段 6：建立性能基准
benchmark 结果稳定且可复现

- 阶段 7：AoS 与 SoA 数据布局
得到可信的 AoS 与 SoA 性能结果

- 阶段 8：基础代码级优化
每项保留的优化都有正确性与性能证据
能解释编译选项对性能的影响

- 阶段 9：编译器优化实验

- 阶段 10：手动 SIMD（可选）

- 阶段 11：多线程并行

- 阶段 12：性能瓶颈分析
