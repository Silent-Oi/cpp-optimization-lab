# Project01 — Batch Oscillator

## 项目简介

Project01 使用大量彼此独立的简谐振子作为第一个 C++ 性能优化实验。

单个简谐振子的模型很简单，因此可以把主要注意力放在：

- 数据如何存储
- 循环如何组织
- 内存如何访问
- 编译优化如何影响结果
- 怎样进行公平的性能比较

项目重点不是物理模型的复杂程度，而是建立最基本的性能分析习惯。

## 物理模型

经典简谐振子的基本形式为：

```text
m x'' + k x = 0
```

也可写成一阶系统：

```text
dx/dt = v
dv/dt = -(k/m)x
```

后续可从最简单的时间积分方法开始，再根据需要比较不同积分方法的数值误差和性能。

## 第一阶段目标

先完成一个容易验证的正确性基线：

- 创建一批独立振子
- 为每个振子保存必要状态
- 推进固定数量的时间步
- 输出少量样本或校验值
- 检查结果是否稳定、合理

这一阶段先不追求性能。

## 第二阶段目标

建立第一个可重复的性能基准：

- 固定振子数量
- 固定时间步数
- 使用 Release 构建
- 多次运行并记录时间
- 防止计算被编译器完全优化掉
- 同时保存结果校验值

## 第三阶段优化实验

建议依次比较：

1. 基础面向对象实现
2. Array of Structures（AoS）
3. Structure of Arrays（SoA）
4. 减少循环内重复计算
5. 不同循环组织方式
6. 不同问题规模下的表现
7. 编译器优化选项的影响
8. 可选：简单多线程版本

每次实验只改变一个主要因素。

## 暂不加入的内容

Project01 初期不需要：

- 复杂继承体系
- 通用物理引擎
- 图形界面
- 第三方数学库
- GPU 计算
- 分布式计算
- 复杂模板元编程

这些内容会分散对基础性能问题的注意力。
